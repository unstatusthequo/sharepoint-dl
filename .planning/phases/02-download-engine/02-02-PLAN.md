Add to `sharepoint_dl/downloader/engine.py`:

- `_make_progress() -> Progress` — creates Rich Progress instance with columns: SpinnerColumn, TextColumn (description), BarColumn(bar_width=None), DownloadColumn, TransferSpeedColumn, TimeElapsedColumn. Follow Pattern 5 from RESEARCH.md.

- `download_all(session, files, dest_dir, site_url, workers=3, progress=None) -> tuple[list[str], list[tuple[str, str]]]`:
  - Creates `JobState(dest_dir)`, calls `state.initialize(files)`
  - Calls `state.cleanup_interrupted(dest_dir)` to delete .part files from prior interrupted run
  - Builds `file_map = {f.server_relative_url: f for f in files}`
  - Gets `pending = state.pending_files()`
  - If `not pending`: return early (all complete)
  - Creates `auth_halt = threading.Event()` and `auth_error: list[AuthExpiredError] = []`
  - If progress provided: creates overall task (`progress.add_task("Overall", total=sum of pending file sizes)`), creates worker tasks (`progress.add_task(f"[worker {i}]", total=0, visible=False)` for i in range(workers))
  - Defines inner `worker(url, worker_id)` function:
    - Check `auth_halt.is_set()` -> return early
    - Get FileEntry from file_map, compute dest_path via `_local_path()`
    - Update state to DOWNLOADING
    - If progress: update worker task description to file name, set total to file size, reset completed to 0, set visible=True
    - Define `on_chunk(n)` callback: updates worker progress task + overall progress task
    - Call `_download_file(session, file_entry, dest_path, site_url, on_chunk=on_chunk)`
    - On success: set state to COMPLETE with sha256 and downloaded_at (ISO 8601 UTC)
    - If progress: hide worker task after completion
    - On AuthExpiredError: set auth_halt, append to auth_error, set state to FAILED with error="auth_expired", re-raise
    - On Exception: set state to FAILED with error=str(e), re-raise
  - `ThreadPoolExecutor(max_workers=workers)`: submit worker for each pending URL with round-robin worker_id
  - `as_completed` loop: on AuthExpiredError, cancel remaining futures and break. On other exceptions, pass (already recorded in state)
  - After executor: if auth_error, raise auth_error[0]
  - Return `(state.complete_files(), state.failed_files())`

```python
@app.command()
def download(
    url: str = typer.Argument(..., help="SharePoint folder URL"),
    dest: Path = typer.Argument(..., help="Local download destination"),
    root_folder: str = typer.Option(
        ...,
        "--root-folder", "-r",
        help="Server-relative path to the folder to download",
    ),
    workers: int = typer.Option(
        3,
        "--workers", "-w",
        min=1, max=8,
        help="Number of concurrent download workers (default: 3)",
    ),
    yes: bool = typer.Option(
        False,
        "--yes", "-y",
        help="Skip confirmation prompt",
    ),
) -> None:
```

Implementation flow:
1. Load session via `load_session(url)` — if None, print red error, exit 1
2. Parse URL via `_parse_sharepoint_url(url)`, use `root_folder` as server_relative_path
3. Validate session via `validate_session(session, site_url)` — if False, print red error, exit 1
4. Enumerate files with `console.status("Scanning folders...")` spinner (reuse pattern from `list_files`)
5. Print confirmation: `"Download {count} files ({_format_size(total)}) to {dest}? [Y/n]"` — use `typer.confirm()`, abort if No. Skip if `--yes`.
6. Create `dest` directory: `dest.mkdir(parents=True, exist_ok=True)`
7. Create Rich Progress via `_make_progress()` (from engine.py)
8. Call `download_all(session, files, dest, site_url, workers=workers, progress=progress)` inside `with progress:` context
9. On success (no failed files): print green summary table — total files downloaded, total size, elapsed time (track with `time.time()`), average speed
10. On failed files: print red Rich Table with columns "File" and "Error" listing each failed file. Then print green count of successful files. Exit with code 1.
11. On AuthExpiredError: print red "Session expired during download. Re-authenticate with 'sharepoint-dl auth <url>' and re-run. Completed files will be skipped on resume." Exit with code 1.
