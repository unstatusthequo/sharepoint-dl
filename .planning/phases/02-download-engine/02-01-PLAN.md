Create `sharepoint_dl/state/job_state.py` with:

- `FileStatus(str, Enum)` with values: `PENDING`, `DOWNLOADING`, `COMPLETE`, `FAILED`
- `JobState` class following the pattern from 02-RESEARCH.md Pattern 3:
  - `__init__(self, dest_dir: Path)` — stores path to `state.json`, creates `threading.Lock`, loads existing state
  - `_load()` — reads state.json if exists, otherwise empty dict
  - `_save()` — atomic write via `.tmp` + `Path.replace()` (POSIX atomic)
  - `initialize(self, files: list[FileEntry])` — populate state for all files not already present; key = `server_relative_url`; include `name`, `size_bytes`, `folder_path`, `status=PENDING`, `sha256=None`, `error=None`, `downloaded_at=None`
  - `set_status(self, server_relative_url: str, status: FileStatus, **kwargs)` — update status and any extra fields (sha256, error, downloaded_at); all operations under `self._lock`; calls `_save()`
  - `pending_files(self) -> list[str]` — returns keys where status is PENDING, FAILED, or DOWNLOADING
  - `complete_files(self) -> list[str]` — returns keys where status is COMPLETE
  - `failed_files(self) -> list[tuple[str, str]]` — returns (url, error) pairs for FAILED entries
  - `cleanup_interrupted(self, dest_dir: Path)` — finds files in DOWNLOADING state, deletes their `.part` files from dest_dir (using folder_path to reconstruct local path), resets status to PENDING
  - `get_entry(self, server_relative_url: str) -> dict | None` — returns the state dict for a file

Create `sharepoint_dl/downloader/engine.py` with:

- `CHUNK_SIZE = 8_388_608` (8 MB)

- `_build_download_url(site_url: str, server_relative_url: str) -> str` — constructs `{site_url}/_layouts/15/download.aspx?SourceUrl={encoded_path}` using `urllib.parse.quote()` with `safe="/:@!$&'()*+,;="`. Follow Pattern 2 from RESEARCH.md.

- `WaitRetryAfter(wait_base)` — custom tenacity wait class. On 429 HTTPError with Retry-After header, returns that value. Otherwise falls back to exponential backoff `min(2 ** attempt, 16)`. Follow Pattern 6 from RESEARCH.md.

- `_download_file(session, file_entry, dest_path, site_url, on_chunk=None) -> str` decorated with `@retry`:
  - `retry=retry_if_exception_type(requests.HTTPError)` — only retries HTTPError, NOT AuthExpiredError
  - `stop=stop_after_attempt(3)`
  - `wait=WaitRetryAfter()`
  - `before_sleep=before_sleep_log(logger, logging.WARNING)`
  - Body: build download URL, create `.part` path (`dest_path.with_suffix(dest_path.suffix + ".part")`), `mkdir(parents=True, exist_ok=True)` on parent
  - `session.get(url, stream=True, timeout=(30, 600))`
  - Check `resp.status_code in (401, 403)` -> raise `AuthExpiredError` BEFORE `raise_for_status()` (mirrors traversal.py pattern exactly)
  - `resp.raise_for_status()` — 429, 5xx become HTTPError, retried by tenacity
  - Stream loop: `for chunk in resp.iter_content(chunk_size=CHUNK_SIZE): if chunk: sha256.update(chunk); fh.write(chunk); on_chunk(len(chunk)) if on_chunk`
  - Size verification: compare `part_path.stat().st_size` to `file_entry.size_bytes`; if mismatch and size_bytes > 0, delete .part, raise ValueError
  - `part_path.rename(dest_path)` — atomic rename on success
  - Return `sha256.hexdigest()`

- `_local_path(dest_dir: Path, file_entry: FileEntry) -> Path` — constructs local path preserving folder structure relative to the root folder. Strip the common prefix from `file_entry.folder_path` to create subdirectories under `dest_dir`.
