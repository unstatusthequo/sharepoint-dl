**Test 1: Basic download (small subset)**
1. Authenticate: `sharepoint-dl auth <your-sharing-url>`
2. List files: `sharepoint-dl list <url> --root-folder <path>`
3. Download a few files: `sharepoint-dl download <url> /tmp/test-download --root-folder <path> --workers 1`
4. Verify: files appear in /tmp/test-download, progress bar was visible, state.json exists in dest dir
5. Check state.json: all downloaded files show status "complete" with sha256 hashes

**Test 2: Resume after interrupt**
1. Start a download of a larger set: `sharepoint-dl download <url> /tmp/test-resume --root-folder <path>`
2. After a few files complete, press Ctrl+C
3. Re-run the same command
4. Verify: already-completed files are skipped (not re-downloaded), only remaining files are attempted
5. Check: no duplicate .part files, state.json shows correct statuses

**Test 3: Progress display**
1. Run a multi-file download with --workers 3
2. Verify: overall progress bar shows "X/Y files (size/total)"
3. Verify: per-worker lines show current file name and progress percentage

**Test 4: Error summary (optional -- hard to trigger naturally)**
- If any file fails during testing, verify it appears in the red error summary at the end
- Verify the tool exits with code 1 (check with `echo $?`)
