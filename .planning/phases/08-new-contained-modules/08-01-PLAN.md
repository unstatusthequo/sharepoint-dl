<task type="auto" tdd="true">
  <name>Task 2: Token bucket throttle module</name>
  <files>sharepoint_dl/downloader/throttle.py, tests/test_throttle.py</files>
  <behavior>
    - TokenBucket(rate_bytes_per_sec=1_000_000) limits throughput to ~1 MB/s
    - TokenBucket.consume(chunk_size) blocks (sleeps) when bucket is empty
    - TokenBucket.consume(chunk_size) returns immediately when tokens are available
    - With no throttle (None passed around), no overhead — callers check `if throttle: throttle.consume(n)`
    - Thread-safe: multiple threads calling consume concurrently stay within rate limit
    - parse_throttle("10MB") returns 10_485_760 (10 * 1024 * 1024)
    - parse_throttle("500KB") returns 512_000 (500 * 1024)
    - parse_throttle(None) returns None
    - parse_throttle("abc") raises ValueError
  </behavior>
  <action>
    Create `sharepoint_dl/downloader/throttle.py`:

    1. `parse_throttle(value: str | None) -> int | None`:
       - None -> None
       - Regex match pattern like `(\d+)(MB|KB|GB)` (case-insensitive).
       - Convert to bytes: KB=1024, MB=1024*1024, GB=1024*1024*1024.
       - Raise ValueError on unparseable input.

    2. `class TokenBucket`:
       - `__init__(self, rate_bytes_per_sec: int)`:
         - `self._rate = rate_bytes_per_sec`
         - `self._tokens = float(rate_bytes_per_sec)` (start full)
         - `self._max_tokens = float(rate_bytes_per_sec)` (cap at 1 second of burst)
         - `self._last_refill = time.monotonic()`
         - `self._lock = threading.Lock()`

       - `consume(self, amount: int) -> None`:
         - Acquire lock.
         - Refill: `elapsed = now - self._last_refill`, add `elapsed * self._rate` tokens, cap at `_max_tokens`, update `_last_refill`.
         - If `self._tokens >= amount`: subtract and return.
         - Else: compute `deficit = amount - self._tokens`, `sleep_time = deficit / self._rate`. Set tokens to 0, release lock, `time.sleep(sleep_time)`.
         - This is the standard token bucket algorithm. Single shared instance across all workers per CONTEXT.md decision.

    Tests in `tests/test_throttle.py`:
    - Unit test parse_throttle with valid/invalid inputs.
    - Test TokenBucket: consume small amount returns instantly (measure time < 0.05s).
    - Test TokenBucket: consume amount > bucket capacity causes measurable sleep.
    - Thread safety test: 4 threads consuming simultaneously, total throughput stays within 2x target rate over 0.5 seconds.
  </action>

  </verify>
