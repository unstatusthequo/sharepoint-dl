<feature>
  <name>ReauthController — check-lock-check session refresh coordinator</name>
  <files>sharepoint_dl/auth/reauth.py, tests/test_reauth.py</files>
  <behavior>
    - trigger() called by first worker: acquires lock, calls on_reauth callback, refreshes cookies on shared session in-place, returns True
    - trigger() called by concurrent workers while lock held: blocks on Event.wait(), returns _last_result when event is set
    - trigger() after MAX_ATTEMPTS (3) reached: returns False immediately without calling on_reauth
    - trigger() when on_reauth raises exception: logs error, sets _last_result=False, sets event so waiters unblock, returns False
    - reset_for_retry() clears _done_event so next 401 round can re-block workers
    - _refresh_cookies() reads session.json via build_session(), clears old cookies on shared session, copies new cookies in-place
    - Each trigger() logs: attempt number, success/failure, elapsed time in seconds
  </behavior>
  <implementation>
    Create ReauthController class in sharepoint_dl/auth/reauth.py per the research pattern:
    - __init__(session, sharepoint_url, on_reauth) where on_reauth is Callable[[str], None] that calls harvest_session
    - MAX_ATTEMPTS = 3 class constant
    - Uses threading.Lock (not RLock) and threading.Event
    - trigger() -> bool: check-lock-check pattern with non-blocking acquire
    - reset_for_retry() -> None: clears done event
    - _refresh_cookies() -> None: loads fresh session from disk, updates shared session.cookies in-place

    Per D-03: update cookies on the EXISTING shared requests.Session — no object swapping.
    Per D-05: max 3 re-auth attempts per download run.
    Per D-07: log each re-auth event with attempt, result, elapsed.
  </implementation>
</feature>
