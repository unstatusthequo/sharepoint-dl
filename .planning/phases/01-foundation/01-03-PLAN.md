  <files>none</files>
  <action>
    Human verification checkpoint. The user must compare the tool's enumeration count against the SharePoint browser UI:
    1. Open the SharePoint sharing URL in your regular browser
    2. Navigate to the root shared folder
    3. Note the total file count shown in the SharePoint UI (check each subfolder if needed)
    4. Run: `uv run sharepoint-dl list "<your-sharepoint-sharing-url>"`

    Also verify:
    7. The per-folder breakdown in the summary table looks correct
    8. The total size reported seems reasonable for forensic evidence files (expect GB range)
    9. No folders were silently skipped — every custodian folder should appear in the output

    If the --root-folder option was needed: Document the server-relative path used, as this will be needed for Phase 2 download configuration.
  </action>
  <verify>User confirms file counts match (tool vs browser). Type "approved" with the file counts, or describe any discrepancies.</verify>
