  <behavior>
    - enumerate_files(session, site_url, server_relative_path) returns List[FileEntry] with all files across all subfolders
    - FileEntry dataclass has: name, server_relative_url, size_bytes, folder_path
    - Pagination: follows d.__next until absent; single-page folders also work
    - Recursion: subfolders discovered via GetFolderByServerRelativeUrl/Folders are pushed to stack
    - Auth expiry: 401/403 response raises AuthExpiredError (custom exception) — NOT retried
    - Transient errors: 429/5xx retried via tenacity (max 3 attempts, exponential backoff)
    - URL encoding: server_relative_url encoded with requests.utils.quote(path, safe="") before embedding in API URL
  </behavior>
  <action>
    1. Create `sharepoint_dl/enumerator/traversal.py`:
       - Define `@dataclass class FileEntry: name: str, server_relative_url: str, size_bytes: int, folder_path: str`
       - Define `class AuthExpiredError(Exception): pass`
       - Define helper `_fetch_page(session, url) -> tuple[list[dict], str | None]`:
         - GET url with headers `{"Accept": "application/json;odata=verbose"}`, timeout=(10, 60)
         - If status 401 or 403: raise `AuthExpiredError("Session expired. Run 'sharepoint-dl auth <url>' to re-authenticate.")`
         - Call `resp.raise_for_status()` for other errors
         - Parse `resp.json()["d"]`, extract `results` list and `__next` URL (None if absent)
         - Decorate with `@retry(wait=wait_exponential(multiplier=1, min=2, max=30), stop=stop_after_attempt(3), retry=retry_if_exception_type(requests.HTTPError), before_sleep=before_sleep_log(logger, logging.WARNING))` from tenacity
         - IMPORTANT: Do NOT retry AuthExpiredError — it must propagate immediately. Use `retry_if_exception_type(requests.HTTPError)` which will not catch AuthExpiredError since it is raised before raise_for_status
       - Define `enumerate_files(session: requests.Session, site_url: str, server_relative_path: str) -> list[FileEntry]`:
         - Uses an explicit stack (list) starting with [server_relative_path]
         - For each folder in stack:
           a. URL-encode the folder path: `encoded = requests.utils.quote(folder_path, safe="")`
           b. Fetch files: initial URL = `{site_url}/_api/web/GetFolderByServerRelativeUrl('{encoded}')/Files?$select=Name,ServerRelativeUrl,Length`
           c. Loop: call `_fetch_page`, append FileEntry for each result, follow `__next` until None
           d. Fetch subfolders: initial URL = `{site_url}/_api/web/GetFolderByServerRelativeUrl('{encoded}')/Folders?$select=ServerRelativeUrl`
           e. Loop: call `_fetch_page`, push each subfolder's ServerRelativeUrl onto stack, follow `__next` until None
       - Filter out system folders: skip subfolders where ServerRelativeUrl contains "/Forms" (SharePoint system folder)

    2. Update `sharepoint_dl/enumerator/__init__.py` to export: `FileEntry`, `enumerate_files`, `AuthExpiredError`

    3. Update `tests/conftest.py`:

         - Folders response (2 subfolders)
         - Empty folder response (no results)
         - 401 response for auth expiry testing

    - `sharepoint-dl download <url> <dest>` raises NotImplementedError with "Phase 2" message

  <action>
    1. Create `sharepoint_dl/cli/main.py`:
       - `app = typer.Typer(no_args_is_help=True, help="SharePoint bulk file downloader for forensic evidence collection.")`

       - `@app.command() def auth(url: str = typer.Argument(..., help="SharePoint sharing URL")):`
         - Import and call `harvest_session(url)`
         - On success: print "[green]Session saved. You can now run 'sharepoint-dl list {url}'[/green]" via rich Console
         - On TimeoutError: print "[red]Authentication timed out. ...[/red]", raise typer.Exit(code=1)

       - `@app.command(name="list") def list_files(url: str = typer.Argument(..., help="SharePoint folder URL")):`
         - Call `load_session(url)` — if None, print error message suggesting `auth` command, raise typer.Exit(code=1)
         - Call `validate_session(session, site_url)` — if False, same error + exit
         - Extract site_url and server_relative_path from the SharePoint URL (parse the URL to get `https://{host}` as site_url and the path portion as server_relative_path)
         - Use `rich.console.Console().status("[bold green]Scanning folders...", spinner="dots")` during enumeration
         - Call `enumerate_files(session, site_url, server_relative_path)`
         - On AuthExpiredError: print re-auth message, exit 1
         - On success: build summary table using `rich.table.Table`:
           - Columns: Folder, Files, Size
           - One row per unique folder_path in results, with file count and human-readable size
           - Final line: "Found {N} files ({X.X GB} total) across {M} folders"
         - Helper function `_format_size(bytes: int) -> str` for human-readable sizes (KB, MB, GB)
         - Helper function `_parse_sharepoint_url(url: str) -> tuple[str, str]` to extract site_url and server_relative_path from a SharePoint sharing URL. The sharing URL format is typically `https://{host}/:f:/g/personal/{path}` or `https://{host}/sites/{site}/Shared%20Documents/{folder}`. For the sharing link format, the server-relative path must be discovered from the authenticated page — for Phase 1, accept an explicit `--root-folder` option as a workaround if the URL cannot be parsed automatically.
