## Unsatisfied Requirements

### DWNL-02: Resume interrupted runs safely

- `sharepoint_dl/state/job_state.py` resets interrupted downloads by deleting every matching `filename.part` under the destination tree.
- This is not path-safe when two different folders contain the same filename.
- Result: resuming one interrupted file can delete another file's partial download.

### VRFY-02: Manifest contains per-file path metadata

- `sharepoint_dl/manifest/writer.py` writes `local_path` from the remote SharePoint folder path, not from the actual destination path used by the downloader.
- This makes the forensic artifact inaccurate.
- The mismatch is guaranteed for `--flat` downloads and possible for normal downloads whenever local layout diverges from the raw SharePoint path.

### ENUM-03: Show total file count before download begins

- The `download` command computes `count` and `total_size`, but it only prints them inside the confirmation branch.
- With `--yes`, transfer begins immediately and the pre-download count is never shown.
- This leaves the integrated download flow only partially compliant with the requirement.

### CLI-03: Clear file-level error summary at end of run

- On `AuthExpiredError` during download, the CLI prints a re-authenticate message and exits immediately.
- The completeness report and failed-download summary table are skipped for that failure mode.
- Normal failure paths do print the summary; auth-expiry currently does not.

## Cross-Phase Integration Gap

### Download -> Report pipeline on auth expiry

The auth-expiry path is detected correctly inside the downloader, but the CLI exits before running the reporting tail:

- no manifest generation
- no completeness report
- no failed-download table

This is the main remaining cross-phase wiring issue.

## Broken Flows

### Resume interrupted run with duplicate filenames

Breaks during cleanup-before-retry. Duplicate names in separate folders can collide because `.part` cleanup is filename-based instead of path-based.

### Download with `--yes`

Breaks at the visibility checkpoint. The user does not see the enumerated file count or total size before downloads start.

### Download interrupted by auth expiry

Breaks at post-failure reporting. The tool exits after the re-auth message without the usual completion/error report.
